# Scripts

Ad-hoc analysis, investigation, and plotting that support the [blog post](../docs/blog.md) and are called by commands in the [reproduction documentation](../docs/blog_reproduction.md).

These are mostly generated by Claude.AI and should not be considered production quality. They are not part of the deliverable pipeline (`workflows/generate_divref.smk`).

Run them via `pixi run`:

```bash
pixi run python scripts/<name>.py
pixi run Rscript scripts/<name>.R
```

Scripts that produce counts emit a machine-readable summary TSV; downstream plots and the blog read from those TSVs, not from hardcoded values.

## Contents

| Script | Purpose |
| --- | --- |
| `compare_haplotypes.py` | Compare the old (DivRef 1.1) vs new `compute_haplotypes` haplotype catalogs from two DuckDB indexes. Partitions into shared / old-only / new-only, checks shared-haplotype sequence equality (partitioned by overlap, not the PASS flag), and compares AC. Pure Python + DuckDB, no Spark. Emits `algo_comparison.summary.tsv`. |
| `compare_haplotypes_venn.R` | Render the old-vs-new haplotype-catalog Venn diagram (`algo_comparison.venn.png`) from `compare_haplotypes.py`'s summary TSV. |
| `compare_divref_gnomad.R` | Plotting for the `compare_divref_gnomad.smk` analysis workflow (DivRef 1.1 vs multiple gnomAD releases). |
| `explain_divref_only_v41.py` | Explain why DivRef 1.1 chr22 variants are absent from the gnomAD v4.1 joint sites table under the AC0-tolerant intersection filter, binning each by reason for absence. Requires Hail. |
| `inspect_threshold_edge_haplotypes.py` | Inspect the chr22 haplotypes the original algorithm emits but the new one does not, against the new algorithm's intermediate Hail tables. Requires Hail + DuckDB. |
| `build_test_reference.py` | Build the masked chr1+chrX GRCh38 test reference committed under `divref/tests/data` (all contigs present at full length, filled with `N` except the chr1/chrX test windows). |
